Responsive Redesign: When2Meet

Enhancing usability and accessibility across devices

Redesign Objectives

The objective of this project is to practice the full workflow of redesigning a webpage. Specifically, the project involves choosing one webpage of choice and analyzing and identifying usability and accessibility problems within that existing web interface, creating visual mockups for different screen sizes, and finally, building a fully responsive website based on these prototypes.

Chosen Web Interface

I selected When2Meet, a popular scheduling tool frequently used to coordinate meetings. Despite its usefulness, When2Meet has clear design flaws—such as the inability to edit schedules once created, ambiguous labeling, and poor accessibility—that negatively impact the overall user experience. By identifying these shortcomings, this redesign aims to create a responsive, intuitive, and inclusive version of the website, enhancing usability across all devices and for all users.

Identifying Usability Problems

Why When2Meet?

I chose this webpage based on my personal experience using it to coordinate meeting times with colleagues. While the site is simple, it lacks thoughtful design in key areas. For example, after creating a schedule with limited availability for the next day, a colleague requested an extension to include the following two days. However, When2Meet does not allow schedule edits, forcing me to generate a new link and ask everyone to re-enter their availability—a tedious and inefficient process. These usability challenges motivated me to redesign the webpage for a more seamless scheduling experience.

Usability Analysis

The analysis below clearly identifies significant usability issues that negatively affect users:

Metrics Problem Description
Efficiency Cumbersome editing process: once a schedule is created, there is no way to modify it. If changes are needed, users will have to create a completely new schedule and ask all participants to re-enter their availability, making this process extremely inefficient.
Learnability Inconsistent navigation pattern making it difficult to get the intuition of how the site is organized and where the flow should be.
Memorability Lack of visual cues such as icons and headings to help users recall where the essential features of the webpage are.
Conceptual Model Ambiguous labeling on key actions and section headings creates a misalignment between user expectations and the intended functionalities of the webpage. For example, the label “No earlier than” is ambiguous regarding the starting time window for scheduling.

Accessibility Analysis

  1. Mouse-Dependent Interaction: The webpage’s scheduling grid can only be accessed via mouse interaction, requiring users to have precise mouse control to click on the days to indicate availability. This design is not keyboard friendly or accessible via alternative input methods.
  2. Inadequate Form Labeling: Elements such as the event name creation lack proper labeling or association with a legend, which can lead to user confusion and incomplete form submissions. The time selection component also does not have a label, further contributing to potential confusion.

Reflection

The accessibility analysis highlights significant and legitimate concerns. The existing interface excludes many potential users, specifically those relying on assistive technologies or using mobile devices. Addressing these issues is essential for inclusive and universally usable design.

Visual Redesign

Visual Design Style Guide

Annotated Mockups & Responsive Decisions

Mobile Mockup (iPhone)

  • Components stacked vertically for intuitive scrolling.
  • Slightly larger fonts and spacing optimized for small screens.

Tablet Mockup (iPad)

  • Similar stacking pattern to mobile with adjusted font size and spacing.
  • Input fields rearrange vertically, ensuring accessibility and readability.
  • Slight reduction in spacing between dropdown options to accommodate narrower widths.

Desktop Mockup

  • Layout splits into a two-column format:
    • Left side: Logo and introductory content for context.
    • Right side: Scheduling interactions (calendar and input fields).

Key Design Improvements

Navigation

  • Clearly defined “Create New Event” and “Edit Existing Event” buttons allow easy mode-switching without ambiguity. Selecting “Edit Existing Event” dynamically adjusts inputs accordingly.

Accessibility Improvements

  • Date and time selections are accessible via keyboard navigation (Tab key).
  • Clear labeling for event name and timezone inputs, meeting accessibility standards.

Responsive Layout Adjustments

  • For smaller screens (tablet/mobile), elements stack logically in chronological order—mode selection, event naming, date selection, and time slots.
  • Desktop layout leverages horizontal space, clearly separating introductory content from interaction components for improved readability.
